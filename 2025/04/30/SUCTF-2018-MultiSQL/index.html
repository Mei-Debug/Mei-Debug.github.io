MySQL预处理

MySQL预处理主要分为变量预处理和执行语句预处理

预处理语句

prepare 语句名称 from 预处理的sql语句

execute 执行预处理语句

drop prepare删除预处理变量

预处理变量

**set @**变量名 = 值; –这里的@是在MySQL中定义变量的一种语法形式（类比php中的$符号）

带变量执行预处理语句

由于需要用到变量，我们的预处理语句需要提前使用占位符，一个?代表一个参数

execute 语句名称 using 参数变量;

这里我们需要注意一下char()这个函数，他不是专门用来转ascii的，只不过默认是ascii，实际上可以配置且功能十分强大

CHAR() 是 MySQL 中的一个字符串函数，用于根据给定的数值代码返回对应的字符。

  • N1, N2, ...：一个或多个整数，代表字符的数值代码
  • USING charset_name：可选参数，指定使用的字符集

CHAR() 函数将每个整数参数解释为字符代码，并返回由这些字符组成的字符串。数值范围应在 0-255 之间。

  • 如果参数为 NULL，则会被忽略
  • 如果参数超出 0-255 范围，结果将为 NULL
  • 默认使用服务器的默认字符集
  • CHAR() 与 CHAR 数据类型是不同的概念
