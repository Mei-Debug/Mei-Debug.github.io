一个命令执行的代码，会进行目录创建然后执行命令，但是有两个过滤函数escapeshellarg和escapeshellcmd，导致我们无法直接进行命令执行。

escapeshellarg() 是 PHP 中一个重要的安全函数，用于在将用户输入传递给 shell 命令时进行安全转义。

escapeshellarg() 函数的作用是：

  • 将字符串转义为一个安全的 shell 参数
  • 在字符串周围添加单引号
  • 转义字符串中已有的单引号

escapeshellcmd() 会转义以下字符：

  • # & ; , | * ? ~ < > ^ ( ) [ ] { } $ \ \x0A \xFF`
  • 单引号和双引号（仅在不成对时）
  • 在 Windows 上，还转义 % 和 !

escapeshellarg处理后先对单引号转义，再用单引号将左右两部分括起来从而起到连接的作用。

127.0.0.1’ -oG经过 escapeshellarg处理后就变成了’127.0.0.1’\‘’ -oG ‘ 也就是发现单引号进行转义，且以它为中心分割为三部分（在两边加单引号） 。

即先对单引号转义，再用单引号将左右两部分括起来从而起到连接的作用。

随后在进经过escapeshellcmd 时变成：’127.0.0.1’\\‘’ -oG \‘也就是将不成对的单引号及任意\进行添加\的操作。

但是由于对\进行了转义，导致多出了一个单引号，然后就会对最后一个单引号进行转义

导致我们后面的-oG可以拼接到命令当中，我们可以利用这个方法进行命令拼接

nmap有一个参数-oG可以实现将命令和结果写到文件
