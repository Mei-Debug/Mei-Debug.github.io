htaccess文件配置及payload

前言

作为文件上传漏洞的重要文件，通过.htaccess可以配置Apache服务器，实现一些修改Apache默认配置。能结合文件上传的原因是，.htaccess覆盖Apache的配置无需重启服务器也可以生效。

.htaccess是一个纯文本文件，它里面存放着Apache服务器配置相关的指令。
.htaccess主要的作用有：URL重写、自定义错误页面、MIME类型配置以及访问权限控制等。主要体现在伪静态的应用、图片防盗链、自定义404错误页面、阻止/允许特定IP/IP段、目录浏览与主页、禁止访问指定文件类型、文件密码保护等。
.htaccess的用途范围主要针对当前目录。

1. .htaccess文件核心组成

.htaccess 文件是一个纯文本文件，通常包含以下类型的指令：

常用指令类型

指令类型 示例 作用
认证与授权 AuthType Basic 密码保护目录
URL 重写 RewriteRule ^old.html$ new.html [R=301] 重定向或美化 URL
错误页面定制 ErrorDocument 404 /404.html 自定义 404 页面
禁止访问 Deny from all 阻止特定 IP 或用户代理访问
MIME 类型设置 AddType application/x-httpd-php .php 定义文件类型处理方式

PHP 配置覆盖 php_value upload_max_filesize 20M 修改 PHP 运行参数（如上传限制）

常见的FLAG

[L] —— Last

停止当前轮次的重写处理。
⚠️ 常见误解：很多人以为 [L] 表示“这是最后一条规则，后面的都不执行”。
真相：[L] 只停止当前轮次。如果 Substitution 是相对路径，Apache 会用重写后的 URL 重新进入重写引擎（新一轮）。

✅ 如果要真正终止，必须配合：

[R] —— Redirect

强制外部重定向（浏览器地址栏变化）。和直接映射到完整url不一样，那个不需要加R，Apache会默认重定向，这个是为了区分映射到某个路由还是重定向

  • R=301 永久重定向，R=302 临时重定向，默认 302
  • 不加 [R] 是内部重写（地址栏不变）

CTF 里出现 [R] 通常用于强制 HTTPS 或者隐藏真实路径。难怪有些题目无论访问什么都跳转到index.php

[F] —— Forbidden

返回 403 禁止访问。

[NC] —— No Case

大小写不敏感匹配。

拦截 admin.php、Admin.php、ADMIN.PHP。

[QSA] —— Query String Append

追加查询字符串，而不是替换。

[E] —— env

[E] 是 RewriteRule 的环境变量标志，用于在重写过程中设置或修改环境变量。它是 Apache 中传递数据的重要机制。

2.Order, Allow, Deny访问控制

虽然我们可以通过RewriteRule去捕获访问的url进行拦截和放行，但是那样写虽然细致，但是太麻烦了，Apache提供给我们可以直接用于访问控制的语句。

这个属于老语法了，Apache2.2及以前默认使用，但是2.2之后使用需要加载mod_access_compat 模块。

Order用于决定Allow和Deny的执行顺序，Allow用于放行，Deny用于禁止

  • Order Allow,Deny = “白名单模式”（最后看Deny，Deny优先级高）
  • Order Deny,Allow = “黑名单模式”（最后看Allow，Allow优先级高）

例1：仅允许192.168.1.100

顺序：先Deny（拒绝所有人）→ 再Allow（允许特定IP）→ Allow覆盖Deny ✅

例2：仅禁止10.0.0.5

顺序：先Allow（允许所有人）→ 再Deny（禁止特定IP）→ Deny覆盖Allow ✅

3.Require访问控制

Require 是 Apache 2.4+ 现代访问控制体系的核心，取代了老旧的 Order、Allow、Deny。

Require没有明确的意思，既可以放行也可以拦截，Require不能单独使用，而是作为一个标记，表示这是Require语句进行访问控制。

简单语句

然后Require后面直接接条件的时候都是放行，然后多个Require语句并列相当于或的关系。

禁止的话使用Require not语句，但是Require not语句不能单独使用。Apache 的语法设计要求：“否定”只能作为组合逻辑的一部分存在，不能作为顶级指令。于是禁止某个ip，在Require语句中表述必须为：其他ip都可以通过，但是这个ip不行。

1.Require容器

为了实现放行和拦截操作，Require语句也需要逻辑，但是相较于Allow和Deny语句那抽象的执行顺序，Require给出了方便的Require容器用来编辑逻辑。

1.<RequireAll> —— 必须满足所有条件（AND）

只有同时满足：内网IP + 不是10.0.0.5 + 不是周日 → 才允许。

2.<RequireAny> —— 满足任一条件即可（OR）

内网IP 或 本机 或 admin用户 → 允许。

3.<RequireNone> —— 不能满足任何条件（NOT）

只要不是这两个IP → 允许（等价于“排除这两个IP”）。

4.嵌套组合

4.处理器指令

到了最重点的地方来了，基本上都是通过将上传的图片马等作为php解析连接shell，但是有一些payload太强了。光会解析图片马还不够。

1.MIME Type

我们在进行文件解析之前得知道什么是MIME Type。

在进行POST传参的时候请求头中就会多出一个Content-Type字段，如果没有这个字段我们请求体的数据就不会被解析，因为服务器不知道该如何解析这些数据。对应的响应头中也会有这个字段，告诉我们浏览器该如何处理返回的响应体。

Content-Type中的值就是MIME Type，MIME Type本身并没有什么强制要求我们可以在请求头中随意改动不会影响文件的属性，而是一种协议被HTTP协议中使用。只是辅助信息，和文件本身没有什么关系。

格式：MIME 类型通常由两部分组成，格式为 主类型/子类型。

  • 主类型（Type）：表示文件的大类，如 text（文本）、image（图像）、application（应用程序数据）等。
  • 子类型（Subtype）：表示具体的文件格式，如 html（HTML 文件）、jpeg（JPEG 图像）、json（JSON 数据）等。

以下是常见的 MIME 类型及其用途：

主类型 子类型 / 示例 用途
text html, css, plain 文本文件 (HTML、CSS、纯文本等)
image jpeg, png, gif 图片文件
audio mp3, wav 音频文件
video mp4, webm 视频文件
application json, pdf, octet-stream 应用程序数据 (JSON、PDF、二进制流等)
multipart form-data 多部分数据 (如表单上传)

示例

处理器

在Apache中会将文件后缀名和MIME Type进行映射，然后根据MIME Type去处理文件，为了处理不同的文件Apache中内置了许多处理器

处理器是服务器内部用来处理特定类型文件的程序：

  • php-script → PHP 解释器
  • cgi-script → CGI 程序
  • server-parsed → SSI（服务器端包含）

MIME 类型和处理器配合工作：MIME 类型告诉“这是什么文件”，处理器决定“谁来执行它”。

2.核心指令

1.AddType —— 添加 MIME 类型映射

语法：AddType MIME类型 扩展名 [扩展名...]

示例

Apache中默认对各种文件后缀有着默认的处理，于是我们就不需要重复将php映射到php后缀的文件上，我们通常将php的MIME类型映射到图片马上面

原理：当访问 .jpg 文件时，Apache 查表发现它的 MIME 类型是 application/x-httpd-php，于是交给 PHP 处理器执行。注意MIME类型不是文件带有的，而是Apache配置的。

2.AddHandler —— 添加处理器映射

语法：AddHandler 处理器名称 扩展名 [扩展名...]

与 AddType 的区别：

  • AddType：告诉“这是什么类型”
  • AddHandler：告诉“谁来处理这个文件”

使用方法

3.SetHandler —— 强制设置处理器

语法：SetHandler 处理器名称

与 AddHandler 的区别：

  • AddHandler：基于扩展名选择性地应用
  • SetHandler：强制所有文件使用指定处理器（无视扩展名）

这是最危险也最强大的指令——上传一个 image.jpg，里面写 PHP 代码，访问它就直接执行。

只对图片文件强制 PHP 解析，更精确。

4.ForceType —— 强制 MIME 类型

语法：ForceType MIME类型

5.RemoveType / RemoveHandler

作用：移除之前设置的 MIME 类型或处理器映射

3.Files 和 FilesMatch 容器

之前的配置基本上都必须基于后缀进行配置，无法精确到文件，Files和FilesMatch分别使用文件名和正则精确匹配文件进行配置。

使用容器匹配文件之后都不需要在指令后面跟上后缀了

<Files>

<FilesMatch>（用正则）

当多个指令冲突时，优先级从高到低：

  1. **SetHandler**（最强，直接指定处理器）
  2. **AddHandler**（指定处理器）
  3. **ForceType**（指定 MIME 类型，间接影响处理器）
  4. **AddType**（指定 MIME 类型）
  5. Apache 默认 MIME 类型映射（mime.types 文件）

结果：.php 文件仍然会执行，因为 SetHandler 优先级更高。

3.expr表达式

在htaccess文件中expr可以实现很多意料之外的操作，.htaccess文件一般都需要配合其他webshell实现服务器解析webshell。然而Apache自然有权限去进行文件读取等操作，而且Apache为了实现某些逻辑，必须通过一些代码去读取信息执行操作。

expr就是htaccess中的表达式语言，很迷你但是可以实现一些基本的逻辑操作。其中file函数可以读取文件内容。

expr 是 Apache 2.4+ 引入的表达式语言，类似编程语言中的条件表达式，可以：

  • 访问请求信息（IP、URI、头等）
  • 调用内置函数
  • 执行字符串操作
  • 进行数值比较
  • 正则匹配
