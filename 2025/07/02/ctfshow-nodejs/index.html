命令执行方式

Node.js 命令执行方法对比表

方法 同步/异步 示例代码 返回结果 安全风险 适用场景 注意事项
execSync 同步 const res = require('child_process').execSync('ls').toString(); Buffer/stdout 🔴 高危 需要立即获取结果的简单命令 绝对禁止传入用户输入
spawnSync 同步 const res = spawnSync('ls', ['-l'], { stdio: 'pipe' }); 对象(stdout/stderr) 🟡 中危 需要精细控制参数的命令 参数需硬编码或严格过滤
exec 异步 exec('ls', (err, stdout) => { console.log(stdout); }); 回调返回stdout 🔴 高危 简单异步命令执行 避免使用shell语法
spawn 异步 const ls = spawn('ls', ['-l']);<br>ls.stdout.on('data', (d) => {...}); 流式输出 🟡 中危 长时间运行或大数据量输出 推荐替代exec
fork 异步 const child = fork('script.js'); IPC通信 🟢 低危 多进程Node.js应用 仅能执行Node脚本
ShellJS 同步/异步 shelljs.exec('ls'); 字符串/stdout 🔴 高危 需要跨平台支持的脚本 本质仍是exec的封装
eval+命令 同步 eval("require('child_process').execSync('ls')");

我们可以使用这些命令执行代码，然后先解析一下

require(‘child_process’).execSync(‘ls’).toString();

require('child_process')

  • Node.js核心模块，提供创建子进程的能力
  • 包含多个执行命令的方法：
    • execSync：同步执行
    • exec：异步执行
    • spawn：更底层的进程生成

由于命令执行结果不是字符串而是二进制

Node.js 中的 Buffer 类是用于直接处理二进制数据的特殊对象，类似于整数数组，但对应 V8 堆外的原始内存分配。

.toString()

  • 将Buffer转换为UTF-8编码的字符串
  • 如果命令输出是文本，可正确显示结果

stdout

  • 作用：spawnSync 方法返回一个对象，这个对象包含关于已完成进程的详细信息。.stdout 是这个对象的一个属性，它专门捕获了子进程执行命令后向标准输出（stdout） 流写入的数据。
  • 说明：在 Unix/Linux 系统中，程序通常有三个标准流：

    • stdout (标准输出 - 1) <- 这里用的就是这个
    • stderr (标准错误 - 2)

web344

最后一个题目竟然不是原型链污染，直接来看代码

req.query.query这个代码获取到请求头的请求参数的query，也就是get传参query。一开始我以为就是发送个json数据即可，但是发现逗号被过滤了。我就没见过Json数据不用逗号隔开的

虽然json数据在URL传输的时候会被编码，但是把%2c也给过滤了

只能去看大佬的payload

首先就是node.js处理req.query.query的时候，它不像php那样，后面get传的query值会覆盖前面的，而是会把这些值都放进一个数组中。而JSON.parse居然会把数组中的字符串都拼接到一起，再看满不满足格式，满足就进行解析，因此这样分开来传就可以绕过逗号了。至于c那个之所以要再进行url编码成%63,就是因为前面的%22，会造成%22c，正好ban了2c，所以c也需要进行url编码。学到了学到了，很有意思的特性。
