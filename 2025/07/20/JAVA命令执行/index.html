JAVA命令执行

前言

java代码也基本可以看懂了，但是还是和以前遇到的问题一样，太多函数和包都不知道，只能先把代码看看

java中命令执行有三种方式

1.java.lang.Runtime

特点：

  • 最传统的方法，直接调用系统命令。
  • 需要手动处理输入/输出流。

可见这个方法是通过getRuntime():获取到Runtime对象，然后调用exec开启子进程并执行命令然后把执行结果返回到Process对象中，后续再通过process.getInputStream()获取到子进程的标准输出

2.java.lang.ProcessBuilder

特点：

  • 更安全的命令执行方式，支持参数化输入，也支持普通输入。
  • 可以设置工作目录、环境变量等。

由于ProcessBuilder可以进程参数命令执行，我们就需要了解一下这个解析方式

  • 规则：
    • 命令和每个参数作为字符串数组的独立元素传递。
    • 参数中的空格、引号、特殊字符（如 |、&）会被视为普通字符，不会触发 Shell 解析。
  • 安全性：
    • 从根本上避免命令注入（因为参数不会被 Shell 解释）。

原理就是将第一个参数（"ls"）作为可执行文件路径或名称（通过 PATH 环境变量查找）也就是我们平时ls，cat等命令都是通过环境变量中的可执行文件使用的

后面参数（"-l", "my dir"）就按顺序传递给程序，由于后续参数不会触发shell解析，而且参数中的各种特殊符号也被视为普通字符，导致我们无法进行命令注入，但是这个实际上也是分情况才能防止命令注入的。

直接调用程序（不通过 Shell）：
参数按原样传递，不会被解析（安全）。

java

  • 输出：hello&calc.exe （纯文本，不会执行 calc.exe）。

通过 Shell 调用（如 cmd.exe 或 /bin/sh）：
参数中的特殊字符 会被 Shell 解析（危险！）。

java

  • 输出：hello 并弹出计算器（& 被解析为命令分隔符）。
场景 示例命令 是否解析特殊字符 安全性
直接调用程序 new ProcessBuilder("echo", "a&&b") ❌ 不解析 ✅ 安全
通过 cmd.exe 调用 new ProcessBuilder("cmd.exe", "/c", "echo a&&b") ✅ 解析 (&&) ❌ 危险
通过 /bin/sh 调用 new ProcessBuilder("/bin/sh", "-c", "echo a&&b") ✅ 解析 (&&) ❌ 危险
