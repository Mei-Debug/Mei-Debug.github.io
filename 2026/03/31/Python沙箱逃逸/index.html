Python沙箱逃逸

沙箱逃逸一直在变着方向出题，但是每次都不会。太菜了。

Python沙箱技术

让用户提交 Python 代码并在服务器上执行，是一些 OJ、量化网站重要的服务，很多 CTF 也有类似的题。为了不让恶意用户执行任意的 Python 代码，就需要确保 Python 运行在沙箱中。

Python 沙箱是一个受限制的执行环境，允许您运行不受信任的 Python 代码，同时限制其访问系统资源和执行危险操作。

然后Python不像PHP和Java，Python设计的初衷就是被用来做脚本的，属于是完全信任执行的代码。Java是将代码放入JVM虚拟机中运行，PHP则是只有请求来了才会运行代码，没有请求时变量内存是不存在的。于是Java和PHP都有有效手段去防范不安全代码的运行。

Python则是Web服务是长期运行不像PHP很难往内存里面写东西，还有就是各种对象都是串在一起的，我们可以通过原型链通过一个对象然后去访问所有内置对象，所以如果放任Python代码去执行那么攻击者可以做的事情就非常多且危害性大。

Python也很难像PHP那样通过disable_function那样直接禁用函数，Python同一个功能可以使用很多个不同的函数去执行，禁用import os等命令执行模块的引入也没用，因为Python的模块实在太多了，很多模块中是包含os模块的。

于是那些需要运行Python代码的网站基本上使用的都是docker容器进行隔离。

题目中的沙箱逃逸

如果所有题目考沙箱逃逸的时候直接使用docker那就没有意思了，根本做不了。于是沙箱的概念其实挺广泛**任何用于限制代码访问系统资源或执行危险操作的技术手段，都可以被广义地称为”沙箱”**。

题目中也基本上是使用各种限制来让我们无法直接进行命令执行等操作，我们就需要通过各种方式去绕过限制，也就叫做沙箱逃逸。

虽然题目一般出的都是jinja2沙箱逃逸，但是Python沙箱逃逸还是需要学一下的。
