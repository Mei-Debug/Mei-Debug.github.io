Metasploit简介

Metasploit全称Metasploit Framework也就是MSF，其中msfconsole是Metasploit Framework 的主控制台程序，它是用户与 Metasploit 交互最常用的方式。

也就是说Metasploit其实不是一个程序，而是一个框架，其集成了：

  • 漏洞利用模块（Exploits）：成千上万个已知漏洞的攻击代码。利用目标系统的已知漏洞进行初步入侵。
  • 载荷生成器（Payloads）：攻击成功后执行的代码（如反弹Shell、添加用户、执行命令）。
  • 辅助工具（Auxiliary）：扫描、枚举、模糊测试等，不会直接利用漏洞。
  • 编码器（Encoders）：用于对payload进行编码，以绕过简单的防护机制，免杀、绕过防火墙。
  • 后渗透工具（Post）：提权、信息收集、横向移动。
  • 免杀工具（Evasion）：和编码器不同，不是通过编码去绕过waf。而是例如生成一些文件格式混淆的免杀样本去绕过一些高级的防护系统。

这些模块下还有很多子模块，但是太多了用到的时候再学。

然后我们在kali中可以通过msfconsole这个程序去和我们的Metasploit框架进行交互。

我们来到kali的/usr/share/metasploit-framework/modules/目录下就可以看的框架的各个模块

2.Exploit对漏洞进行利用

我们可以根据search 匹配到的exploit对靶机进行攻击，我们选择对应的exploit后可以通过info来查看漏洞和payload的具体信息

我们可以凭借info的输出来判断漏洞的利用程度以及攻击方式，和我们需要完成的一些条件，比如必须登录账号等。

我们也可以不用info来输出全部信息，而是通过show options , show taregts等来指定信息输出。

我们在满足参数和攻击平台的要求后，我们就需要选择payload，当然不选的话会直接使用默认的payload，我们可以通过show payloads命令查看可以使用的payload

我们选择几条来分析，这些主要分为三类分别是Meterpreter类，shell类，一次性执行类类。

然后那些描述我们主要是关系shell是正向连接还是反向连接，其中区别为

  • 正向连接：在msf中通常是指，bind_tcp，定义–攻击机主动发起连接靶机，连接到目标靶机特定的一个端口，目标机器也需要开启一个监听端口，比如tcp监听
  • 反向连接：在msf中通常是指，reverse_tcp，定义–更常用，通过设置reverse_tcp攻击载荷可以轻松实现反弹，监听只需要在本机开启监听端口。反弹shell就是反向连接，通过目标靶机去连接攻击机。
  • 主要区别就是公网和局域网的不同利用，正向连接通常攻击机用在局域网中不能访问 外网靶机，反向连接需要两个机器都需要能互相ping

我们使用一个反向连接来测试

3.Meterpreter进行攻击

我们漏洞利用成功后可以使用Meterpreter模块进行攻击，但是Meterpreter的命令实在太多了，不同的payload可以使用的命令也不同，但是核心命令基本上相同，于是我就记一下核心命令，等到要用的时候直接?可以查看当前可用命令

命令 别名 说明
? - 显示帮助菜单
background bg 将当前会话挂入后台
bgkill - 终止正在运行的后台脚本
bglist - 列出所有后台运行的脚本
bgrun - 以后台线程方式运行一个 Meterpreter 脚本
channel - 查看或控制活动通信频道
close - 关闭指定 channel
detach - 分离会话（适用于 HTTP/HTTPS 隧道）
disable_unicode_encoding - 禁用 Unicode 编码（用于兼容旧系统）
enable_unicode_encoding - 启用 Unicode 编码
exit / quit 互为别名 结束当前 Meterpreter 会话
get_timeouts - 获取当前会话超时设置
guid - 获取会话唯一 GUID
help - 显示帮助信息（同 ?）
info - 显示 Post 模块详细信息
irb - 在当前会话中启动交互式 Ruby shell
load use（不推荐） 加载扩展模块（如 powershell, python）
machine_id - 获取目标机器在 MSF 中的唯一 ID
migrate - 迁移进程（提升稳定性，避免崩溃）
pivot - 管理内网流量转发监听器
pry - 启动 Pry 调试环境（高级调试）
read - 从 channel 读取数据
resource - 执行本地存储的 .rc 脚本文件
run - 运行 Meterpreter 脚本或后渗透模块
secure - 重新协商 TLV 数据包加密机制
sessions - 快速切换到其他会话
set_timeouts - 设置会话超时时间（秒）
sleep - 让 Meterpreter 静默一段时间后重连
ssl_verify - 修改 SSL 证书验证行为
transport - 管理传输方式（如 http → https 切换）
uuid - 获取当前会话的 UUID
