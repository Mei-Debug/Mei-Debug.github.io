sql注入过滤绕过技巧

总结一下常见的绕过方式，以及sql的一些特性。

我以最常见的联合注入语句来看一下哪些会被过滤

为此我还去把很久没用的sqliab靶场给用上了。

1.符号过滤

1.空格过滤

语句的关键词之间需要使用空格来分隔。

主要的绕过思路就是如果只是单独过滤了空格，可以使用其他sql语句中支持的空白字符用于分隔比如Tab，还有就是可以使用注释符进行分隔。

空白字符绕过空格，MySQL中指定了空白字符可以用于分隔关键字不只是空格

MySQL官方文档中明确将其列为空白符（Whitespace）：

  1. 空格：0x20 （%20 或 +）
  2. 制表符 (Tab)：0x09 （%09）
  3. 换行符 (Newline)：0x0A （%0a）
  4. 垂直制表符 (Vertical Tab)：0x0B （%0b）
  5. 换页符 (Form Feed)：0x0C （%0c）
  6. 回车符 (Carriage Return)：0x0D （%0d）

关于在其他博客上看到的%00空字节实际上是不能作为分隔符的，想想也知道分隔符虽然是空白字符，但是不会是空字节这么基础的字符。空字节的作用主要是用来截断字符串，使用在比如数组，字符串等场景用于截断，比如写了这种C语言代码（每种语言的储存方式都不一样，C语言中用于截断）

如果你输入admin%00xxxx，strcpy会把%00当作结束符，只复制admin进去，后面的xxxx被截断了。

在学文件上传的时候就遇到过%00截断，当我们上传文件名为shell.php%00.jpg的时候在检测文件名的时候不会解析空字符截断，但是在保存文件的时候会进行截断然后保存为shell.php，这可能是在底层代码中的问题，php5.3.4版本已经修复了。

注释分隔关键字

空白字符起的就是一个分隔作用，只需要有分隔效果就可以绕过空格了。

其中内联注释符比多行注释多了一个!，其不仅可以作为分隔符使用，MySQL规定/*!union*/实际上就是 union，里面的内容会被作为sql代码执行。

关键特性：

  • /*!union*/ 实际上就是 union
  • 可以用来绕过简单的关键字过滤（WAF可能不认识这种语法）
  • 可以放在关键字中间：uni/*!*/on → 变成 union

/*!union*/ 被解析为 union 是因为它根本就不是注释，而是 MySQL 的条件编译指令。它的设计初衷是为了兼容性，但后来也成了 Web 应用防火墙（WAF）绕过的常见手法。

这种绕过方法还在25年的国赛中考到了，但是其实不知道后端是如何过滤的，如果关键字过滤的话条件编译注释也无法绕过吧，莫非是后端先忽略了注释里面的内容再进行过滤，但是执行sql语句的时候又把注释加上了。

括号绕过空格

括号绕过空格有很多请况都可以使用，我们来看一下无空格报错注入

我们可以看到select from where这些语句后面都是可以不使用空格，使用括号也可以正常执行。于是我们只需要知道哪些请况不能使用括号代替空格。

经过一番总结，我发现括号中可以包含各种语句的参数，之前是使用空格进行分隔函数与参数，实际上可以使用括号包裹参数绕过空格，但是如果遇到的是固定搭配的两个关键字就不能绕过空格了。

比如union select是由union和select两个函数组成的，可以使用括号绕过空格

2.引号过滤

如果把我们输入的引号过滤了其实挺致命的，因为sql注入的第一个条件就是可以闭合原来的语句然后再进行操作。

  • 如果参数是数字类型，那么即使过滤了引号也不会影响到我们闭合。
  • 还有就是可能不是直接匹配然后过滤了引号，而是利用转义函数进行转义在GBK编码的请况下可以尝试使用宽字节注入，最常见的就是%df'当引号被转义时会变为%df%5c'其中%df%5c为宽字节指向的是中文的’運’字。
  • 某些代码会将我们的输入先进行一次过滤，再进行一次类似url解码，于是我们就可以通过对引号进行二次url编码进行绕过

然后就是我们遇到一条sql语句我们可以传入两个where的参数的时候，我们可以利用一个技巧，传入转义符号，同时转义第一个参数和第二个参数的一个引号，这样剩下的两个引号就可以配合起来

这个语句虽然不能执行，主要是因为password后面多出了一个被转义了的引号，但是之前做题的时候遇到了其中帮我们过滤掉了\'

理想状态是这样的

我们做题目的时候遇到的一般不会让我们闭合不了，简单的题目会使用数字型的参数意思一下，然后主要考我们的是用各种方式去不使用引号的请况下用各种方式完成注入。

我们正常注入有很多地方需要使用到引号，主要是表名，以及where语句等

然后尝试了一下反引号可不可以替换掉引号，发现在where语句这些指定字符串的地方不能使用，看来反引号还是一个可有可无的存在。

3.逗号过滤

逗号在sql语句中的作用很重要，根据功能划分可以简单分为两类：

  • 在增删改查等操作中用于分隔列名，比如union select 1,2,3。
  • 函数中用于分隔参数，比如if，substr，limt这些函数都需要使用逗号分隔参数

我们需要通过join语句在进行联合查询的同时，拼接三列来实现类似的效果。

4.逻辑运算符过滤

sql中的逻辑运算符就只有<>=，绕过方法主要是通过sql内置的函数达到同样的效果

=绕过

等号主要有两个用处，一个是盲注，一个是万能密码。

我们最早接触的万能密码是'or 1=1 #，其实如果是达到效果直接'or 1 #也可以，使用1=1的原因是这个属于布尔表达式，在各种环境下都会返回true。在某些环境中除0以外的数字和字母都会被自动解析布尔为true，但是如果没有进行自动解析则会失效。

但是这种请况基本上没有遇见过，如果使用万能密码时=被过滤了，我们就直接来个1就行了

1.and和or过滤

or常被用做万能密码，and和or基本上是布尔盲注的一个基础条件。时间盲注有时还需要在查询前使用and，不然盲注代码就有可能不会被执行。

and和or在代码中可以分别使用&&和||替代，除了这两个符号之后就不存在等效替换了。
